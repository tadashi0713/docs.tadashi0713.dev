# ポリシーの設定

一部のポリシータイプは設定可能です。これにより、そのポリシータイプに対して独自の設定パラメータを持つ新しいカスタマイズバージョンを作成できます。また、一時的に無視したい場合はポリシーを無効にすることができ、必要に応じてポリシーを完全に削除することもできます。

<Callout type='info'>
  ポリシーを削除またはカスタマイズすると、デフォルトのポリシー設定に対する過去の評価結果は削除されます。
</Callout>

## ポリシーを追加する

新しいポリシーを追加するには、カスタマイズしたいポリシータイプを選択します。すべてのカスタムポリシーは、ベースとしてポリシータイプを使用します。

新しいポリシーの表示名や説明を編集して、ポリシーの準拠・非準拠状態をよりわかりやすく伝えることができます。ポリシータイプの名前自体は変更できず、表示名のみ変更可能です。

ポリシーで利用できる設定パラメータは、編集対象のポリシータイプによって異なります。詳細については、[ポリシータイプ](/scout-ja/policy#ポリシータイプ)を参照してください。

ポリシーを追加する手順：

1. Docker Scout ダッシュボードの[ポリシーページ](https://scout.docker.com/reports/policy)に移動します。
2. **Add policy** ボタンを選択して、ポリシー設定画面を開きます。
3. ポリシー設定画面で、設定したいポリシータイプを見つけ、**Configure** を選択してポリシー設定ページを開きます。

    - **Configure** ボタンがグレー表示されている場合は、現在のポリシーに設定可能なパラメータがありません。
    - ボタンが **Integrate** と表示されている場合は、ポリシーを有効化する前にセットアップが必要です。**Integrate** を選択すると、統合のセットアップガイドに移動します。

4. ポリシーのパラメータを更新します。
5. 変更を保存します：

   - **Save policy** を選択すると、変更が反映され、現在の組織に対してポリシーが有効になります。
   - **Save and disable** を選択すると、ポリシーを無効化したまま設定内容が保存されます。

## ポリシーを編集する

ポリシーを編集すると、設定をゼロから作成することなく変更できます。これは、要件や組織のコンプライアンス目標の変更に応じてポリシーパラメータを調整する際に便利です。

ポリシーを編集する手順：

1. Docker Scout ダッシュボードの[ポリシーページ](https://scout.docker.com/reports/policy)に移動します。
2. 編集したいポリシーを選択します。
3. **Edit** ボタンを選択します。
4. ポリシーのパラメータを更新します。
5. 変更を保存します。

## ポリシーを無効化する

ポリシーを無効化すると、そのポリシーに対する評価結果は非表示となり、Docker Scout ダッシュボードや CLI に表示されなくなります。ポリシーを無効化しても過去の評価結果は削除されません。そのため、再度ポリシーを有効化した際には以前の評価結果が再び表示されます。

ポリシーを無効化する手順：

1. Docker Scout ダッシュボードの[ポリシーページ](https://scout.docker.com/reports/policy)に移動します。
2. 無効化したいポリシーを選択します。
3. **Disable** ボタンを選択します。

## ポリシーを削除する

ポリシーを削除すると、そのポリシーの評価結果も削除され、Docker Scout ダッシュボードや CLI に表示されなくなります。

ポリシーを削除する手順：

1. Docker Scout ダッシュボードの[ポリシーページ](https://scout.docker.com/reports/policy)に移動します。
2. 削除したいポリシーを選択します。
3. **Delete** ボタンを選択します。

## 削除したポリシーの復元

ポリシーを削除した場合は、[ポリシーの追加](#ポリシーを追加する)の手順に従って再作成できます。ポリシー設定画面で、再作成したい削除済みのポリシーで **Configure** を選択します。

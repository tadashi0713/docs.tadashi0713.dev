# 統合機能

agentエージェントは、使用目的に応じてさまざまな環境と統合できます。各統合タイプは特定の目的を果たします。

## 統合タイプ

### ACP - エディタ統合

cagentエージェントをエディタ（Neovim、Zedなど）の中で直接実行します。エージェントはエディタのファイルコンテキストを参照し、エディタのインターフェースを介してファイルを読み取り、修正することができます。エディタに組み込まれたAIコーディングアシスタントが必要な場合は、ACPを使用してください。

セットアップ手順については、[ACP統合](/ai/cagent/integrations/acp/)を参照してください。

### MCP - ツール統合

Claude DesktopやClaude CodeなどのMCPクライアントに対して、cagentエージェントをツールとして公開します。エージェントはクライアントのツールリストに表示され、クライアントは必要に応じてそれらを呼び出すことができます。Claude Desktop（または他のMCPクライアント）から専門特化したエージェントにアクセスさせたい場合は、MCPを使用してください。

セットアップ手順については、[MCP統合](/ai/cagent/integrations/mcp/)を参照してください。

### A2A - エージェント間通信（Agent-to-Agent）

cagentエージェントをHTTPサーバーとして実行し、他のエージェントやシステムがAgent-to-Agentプロトコルを使用して呼び出せるようにします。エージェントは、他のシステムがネットワーク経由で検出し、呼び出すことができるサービスになります。マルチエージェントシステムを構築したい場合や、エージェントをHTTPサービスとして公開したい場合は、A2Aを使用してください。

セットアップ手順については、[A2A統合](/ai/cagent/integrations/a2a/)を参照してください。

## 適切な統合の選択

| 機能 | ACP | MCP | A2A |
| :--- | :--- | :--- | :--- |
| **ユースケース** | エディタ統合 | ツールとしてのエージェント | エージェント間の呼び出し |
| **トランスポート** | stdio | stdio/SSE | HTTP |
| **検出方法** | エディタプラグイン | サーバーマニフェスト | エージェントカード |
| **最適の用途** | コード編集 | ツール統合 | マルチエージェントシステム |
| **通信方向** | エディタがエージェントを呼ぶ | クライアントがツールを呼ぶ | エージェント間 |

コードを書いている最中にエディタにエージェントを組み込みたい場合は、**ACP**を選択してください。Claude Desktop（または他のMCPクライアント）から専門のエージェントをツールとして呼び出せるようにしたい場合は、**MCP**を選択してください。エージェント同士がHTTP経由で互いに呼び出し合うマルチエージェントシステムを構築する場合は、**A2A**を選択してください。

# Kiro サンドボックス

このガイドでは、サンドボックス環境での Kiro の認証、設定、および使用方法について説明します。

公式ドキュメント（英語）: [Kiro CLI](https://kiro.dev/docs/cli/)

## クイックスタート

プロジェクトディレクトリに対してサンドボックスを作成し、Kiro を実行するには：

```shell
$ docker sandbox run kiro ~/my-project
```

ワークスペースのパラメータは任意です。省略した場合は現在のディレクトリがデフォルトになります：

```shell
$ cd ~/my-project
$ docker sandbox run kiro
```

初回実行時、Kiro はデバイスフローを使用した認証を求めます。

## 認証

Kiro はデバイスフロー認証を使用します。これはWebブラウザを通じた対話型ログインを必要とする方式です。この方法により、APIキーを直接保存することなく安全な認証が可能になります。

### デバイスフローによるログイン

初めて Kiro を実行すると、認証を促すメッセージが表示されます：

1. Kiro がURLと確認コードを表示します。

2. WebブラウザでそのURLを開きます。

3. 確認コードを入力します。
    
4. ブラウザ上で認証フローを完了させます。

5. ターミナルに戻ると、Kiro が自動的に処理を続行します。

認証セッションはサンドボックス内に保持されるため、サンドボックスを破棄して再作成しない限り、繰り返しログインする必要はありません。

### 手動ログイン

以下のコマンドで、手動でログインフローを開始することもできます：

```shell
$ docker sandbox run <sandbox-name> -- login --use-device-flow
```

このコマンドは、コーディングセッションを開始せずにデバイスフロー認証のみを実行します。

### 認証の永続性

Kiro は認証状態をサンドボックス内の `~/.local/share/kiro-cli/data.sqlite3` に保存します。このデータベースは、サンドボックスが存在する限り保持されます。サンドボックスを削除した場合、再作成時に再度認証が必要になります。

## 設定

Kiro は最小限の設定で動作します。エージェントはデフォルトで「全ツール信頼（trust-all-tools）」モードで動作するため、繰り返しの承認なしにコマンドを実行できます。

### 実行時にオプションを渡す

サンドボックス名と `--` セパレーターの後に Kiro CLI オプションを記述します：

```shell
$ docker sandbox run <sandbox-name> -- chat --trust-all-tools
```

`chat --trust-all-tools` コマンドにより、承認プロンプトが無効化された状態で Kiro が起動します。

## ベースイメージ

テンプレート: `docker/sandbox-templates:kiro`

Kiro は対話型のデバイスフローを通じて認証を管理します。認証データベースはサンドボックスの再起動後も保持されます。デフォルトで chat --trust-all-tools が付与された状態で起動します。

独自のイメージを構築したい場合は、[カスタムテンプレート](/ai/sandboxes/templates)を参照してください。

# カスタムシェル

このガイドでは、エージェントのカスタムインストールや開発のための最小限の環境である「Shell（シェル）」サンドボックスについて説明します。他のエージェント専用サンドボックスとは異なり、Shell にはエージェントのバイナリがプリインストールされていません。その代わり、任意のエージェントやツールを自由にインストール・設定できるクリーンな環境を提供します。

## クイックスタート

サンドボックスを作成し、シェル環境を起動します：

```shell
$ docker sandbox run shell ~/my-project
```

ワークスペースのパラメータは任意です。省略した場合は現在のディレクトリがデフォルトになります：

```shell
$ cd ~/my-project
$ docker sandbox run shell
```

これにより、サンドボックス内で bash ログインシェルが起動します。

## 主なユースケース

Shell サンドボックスは、主に以下の目的で使用されます：

- カスタムエージェントのインストール

    Docker Sandboxes で公式にサポートされていないエージェントをインストールできます。環境にはパッケージマネージャーと開発ツールが含まれており、任意のソフトウェアを導入可能です。

- エージェントの開発

    プライベートDockerデーモンを備えた隔離環境で、カスタムエージェントの実装や修正をテストできます。

- 手動設定

    標準テンプレートではサポートされていない、複雑なセットアップ要件やカスタム認証フローを必要とするエージェントを構成できます。

- トラブルシューティング

    コマンドを手動で実行したり、サンドボックス環境を直接検査したりすることで、エージェントの問題をデバッグできます。

## 認証

Shell サンドボックスは、プロキシによる認証情報の注入を使用します。サポートされているプロバイダー（OpenAI, Anthropic, Google, GitHub など）への API リクエストに対して、プロキシが認証情報を自動的かつ透過的に注入します。

ホスト側のシェル設定ファイルに API キーを設定してください：

```shell
export OPENAI_API_KEY=sk-xxxxx
export ANTHROPIC_API_KEY=sk-ant-xxxxx
export GOOGLE_API_KEY=AIzaSyxxxxx
export GH_TOKEN=ghp_xxxxx
```

変更を適用する：

- シェル設定を読み込む： `source ~/.bashrc` （または `~/.zshrc`）

- Docker Desktop を再起動して、デーモンに環境変数を認識させる

- サンドボックスを作成して実行する：

```shell
$ docker sandbox create shell ~/project
$ docker sandbox run <sandbox-name>
```

プロキシはホスト環境から認証情報を読み取り、API リクエストに自動的に注入します。認証情報がサンドボックス内に保存されることはありません。


## エージェントのインストール

シェル・サンドボックス内に入ったら、標準的なインストール方法でエージェントを導入してください。

### 例：Continue のインストール

[Continue](https://continue.dev) は AI コードアシスタントです。Node.js がプリインストールされているため、直接インストール可能です：

```shell
$ npm install -g @continuedev/cli
$ cn --version
1.5.43
```

コンテナ化されたエージェントや複雑なセットアップが必要な場合は、対話形式でインストールする代わりに、シェルテンプレートをベースにした[カスタムテンプレート](/ai/sandboxes/templates)の作成を検討してください。

## コマンドの実行

`--` セパレーターの後にシェルオプションを渡すことで、コマンドを直接実行できます：

```shell
$ docker sandbox run <sandbox-name> -- -c "echo 'Hello from sandbox'"
```

## ベースイメージ

テンプレート: `docker/sandbox-templates:shell`

Shell テンプレートは、プリインストールされたエージェントを含まないベース環境を提供するため、エージェントの手動インストールに最適です。

独自のイメージを構築したい場合は、[カスタムテンプレート](/ai/sandboxes/templates)を参照してください。

このテンプレートは非常に最小限の構成であるため、あらゆるカスタムエージェント構築のベースとして適しています。

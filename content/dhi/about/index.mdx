# 概要

Docker Hardened Images（DHI）は、現代のソフトウェアサプライチェーンにおけるセキュリティ、コンプライアンス、信頼性を目的として設計されています。

このセクションでは、DHI が標準的なベースイメージやアプリケーションイメージと何が異なるのか、どのように構築・テストされているのか、そしてコンテナ化されたワークロードのセキュリティ確保において Docker とユーザーがどのように責任を分担するのかについて説明します。

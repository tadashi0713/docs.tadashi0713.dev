# シームレスな統合

Docker Hardened Images（DHI）は、既存の開発およびデプロイメントワークフローに無理なく統合できるよう設計されており、セキュリティの強化と使いやすさの両立を実現します。

## Docker Hub でイメージを確認する

Organization が [サインアップ](https://www.docker.com/products/hardened-images/#getstarted) を完了すると、チームは Docker Hub 上で DHI カタログ全体を直接確認できるようになります。開発者やセキュリティチームは以下のようなことが可能です:

- 利用可能なイメージや言語／フレームワークのバリアントを確認する

- サポートされているディストリビューションを把握する

- 開発用バリアントと実行時バリアントの違いを比較する

各リポジトリには、サポートされているタグ、ベースイメージの構成、イメージ固有のドキュメントなどのメタデータが含まれており、プロジェクトに最適なバリアントを選ぶのに役立ちます。

## CI/CD ワークフローで DHI を使用する

Docker Hardened Images（DHI）は、Dockerfile を用いて構築される任意の CI/CD パイプラインにおいて、ベースイメージとしてそのまま利用できます。

GitHub Actions、GitLab CI/CD、Jenkins、CircleCI など、チームがすでに使用している自動化プラットフォームとも簡単に統合できます。

## DevSecOps スタックに適合する設計

Docker Hardened Images は、既存の DevSecOps ツールチェーンとシームレスに連携するよう設計されています。

DHI は、スキャニングツール、コンテナレジストリ、CI/CD システム、ポリシーエンジンなど、チームがすでに使用している各種ツールと連携可能です。

Docker は幅広いエコシステムパートナーと連携しており、DHI が既存のワークフローやツールとそのまま動作することを保証しています。

これらのパートナーにより、スキャン強化、メタデータ検証、コンプライアンスインサイトの提供がパイプライン内で直接可能になります。

すべての DHI には以下が含まれています:

- 署名付きソフトウェア部品表（SBOM）

- CVE データ

- 脆弱性悪用可能性情報（VEX ドキュメント）

- SLSA Build Level 3 に準拠したビルドプロビナンス

これらのメタデータは署名され、構造化されているため、ポリシーエンジンやダッシュボードに取り込んで監査やコンプライアンスワークフローに活用することが可能です。

## 任意のレジストリ経由で配布する

DHI（Docker Hardened Images）は、Docker Hub 上の貴社の名前空間にミラーされます。そこから、任意の OCI 準拠レジストリにプッシュすることも可能です。たとえば:

- Amazon ECR

- Google Artifact Registry

- GitHub Container Registry

- Azure Container Registry

- Harbor

- JFrog Artifactory

- その他の OCI 準拠オンプレミス／クラウドレジストリ

ミラーリングにより、チームはポリシーやビルドシステムを壊すことなく、好みの場所からイメージを pull できるようになります。

## まとめ

Docker Hardened Images は、開発から CI、スキャン、デプロイに至るまで、既存のツールと統合されます。DHI は次のような特徴を備えています：

- 標準的な Docker ツールやパイプラインとそのまま連携可能

- 主要なスキャナやコンテナレジストリに対応

- 既存のコンプライアンスシステムに統合できるセキュリティメタデータを含む

つまり、エンジニアリングワークフローを中断することなく、より強力なセキュリティ管理を導入できます。

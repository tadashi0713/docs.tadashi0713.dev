# 柔軟なリポジトリ単位の価格体系

Docker Hardened Images は、セキュリティやコンプライアンスだけでなく、運用面およびコスト面での効率性も重視して設計されています。

リポジトリ単位で課金されるモデルにより、使用量とコストを正確にコントロールできます。

## 自由に選べるリポジトリミラーリング

Docker Hardened Images では、リポジトリ単位でミラーを行うことで、対応するすべてのタグ、バリアント、バージョンにアクセスできます。

どのリポジトリをミラーするかは、ニーズに応じて柔軟に選択可能です。

この柔軟性により、プロジェクトの進行に応じて環境を立ち上げたり、ランタイムを統合したり、コストを長期的に管理したりといった運用が可能になります。

個別のイメージや pull 数ごとの課金を気にする必要はありません。

## すべてのバリアントとバージョンにアクセス可能

Docker Hardened Image のリポジトリをミラーすると、そのリポジトリ内でサポートされているすべてのタグにアクセスできます。

これには、複数のバージョン、ベースディストリビューション（Alpine や Debian など）、および開発用／実行時バリアントが含まれます。

各ユースケースに最適なタグを追加コストなしで自由に選択できます。

この柔軟な仕組みにより、チームはコストや課金構造に縛られることなく、セキュアなイメージを安心して採用できます。

## チーム全体でのアクセス共有

一度リポジトリをミラーすれば、Organization 内の誰でもそのイメージを pull、検証、スキャン、実行することができます。

利用量に応じた追加料金は発生しません。

必要なリポジトリだけをミラーし、チームは自由に活用できます。

## プラットフォームチームにとってのコスト効率

この課金モデルは、プラットフォームチームやセキュリティチームにとって予算管理をシンプルにします。

個別のイメージやタグ単位で利用状況を追跡するのではなく、管理対象のリポジトリ単位でコストを把握できるため、セキュリティポリシーの適用、チームのアクセス管理、コスト管理を一元化できます。

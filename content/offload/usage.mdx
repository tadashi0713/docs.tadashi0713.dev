# Docker Offload の利用状況と課金

> [!NOTE]
> 
> Docker Offload Beta で付与されたすべての無料トライアルクレジットは、付与されてから 90 日後に失効します。
> クレジット失効後も Docker Offload を継続利用するには、[Docker Home Billing](https://app.docker.com/billing) でオンデマンド利用を有効化してください。

## Docker Offload の課金

Docker Offload の課金情報は、[Docker Home Billing](https://app.docker.com/billing) 内の **Docker Offload** ページで確認・設定できます。

このページでは次の操作が可能です：

- 含まれている予算を確認

- クラウドリソースの料金を確認

- オンデマンド利用の有効化／無効化

- 支払い方法の追加または変更

課金に関する一般的な情報については、[Billing](https://docs.docker.com/billing/) を参照してください。

## Docker Offload overview

Docker Home の **Docker Offload overview** ページでは、あなたやチームがクラウドリソースを使ってどのようにコンテナをビルド・実行しているかを可視化できます。

**Overview** ページを表示するには：

1. [Docker Home](https://app.docker.com/) にサインインします。

2. Docker Offload を管理したいアカウントを選択します。

3. **Offload** > **Overview** を選択します。

以下のセクションでは、**Overview** ページで利用できるウィジェットについて説明します。

### Offload minutes

このウィジェットは、時間の経過に伴うオフロード分数（Offload minutes）の合計を表示します。
Offload minutes とは、Offload 環境でビルドやコンテナを実行するのに費やした時間を表します。

このチャートを利用して次のことが可能です：

- オフロード利用状況の推移を追跡

- CI の変更やビルド問題を示す可能性のある使用量の急増を検知

- サブスクリプションの上限に対する利用状況を見積もり

### ビルドキャッシュの利用状況

このウィジェットは、すべてのビルドにおけるキャッシュの再利用データを表示し、Docker Offload がビルドキャッシュをどれだけ効率的に利用しているかを把握できます。

次のような洞察が得られます：

- キャッシュヒットとミスの割合

- キャッシュレイヤーの再利用によって節約された推定ビルド時間

- Dockerfile やビルド戦略を調整することでキャッシュ効率を改善できるポイント

### ビルドが多いリポジトリ（Top repositories built）

このウィジェットは、Docker Offload において最もビルドアクティビティの多いリポジトリを表示します。
どのプロジェクトが最もクラウドリソースを消費しているか、またどれだけ効率的にビルドされているかを把握できます。

集約されたメトリクスとリポジトリごとの詳細が含まれており、包括的なビューを提供します。

このウィジェットを使って次のことが可能です：

- ビルドのホットスポットを特定: どのリポジトリが最もビルド時間とリソースを消費しているかを確認

- トレンドを検知: プロジェクト全体でビルドアクティビティがどのように推移しているかを監視

- 効率性を評価: キャッシュ再利用の恩恵を最も受けているリポジトリを確認

- 改善対象を特定: キャッシュヒット率が低い、または失敗率が高いリポジトリを最適化対象としてフラグ付け

### 上位 10 イメージ（Top 10 images）

このウィジェットは、Docker Offload の実行セッションで使用された 上位 10 個のイメージ を表示します。
どのイメージが最も頻繁に利用されているかを把握でき、チームのコンテナ利用パターンを理解するのに役立ちます。
